# CS255-System-Analysis-and-Design
This repository contains my portfolio submission for CS 255.

The DriverPass project is for a faux-client DriverPass who requested a system be made for their driving school. They wanted a web-focused system to hold their schedules, online classes, and purchasing requirements, amongst other things.

I think that the diagrams created in the System Design Document are my strongest piece of this project. I belive that I designed them to be easily used as reference when making that actually system architecture, and made sure not to include anything that wouldn't be helpful during the development phase of the SDLC.

If I could revise anything, I think I would chose to put more into the write up conclusion at the end of the System Design Document. It is sufficient, but I could have gone in better detail.

I interpret user's needs by listening to the interview with the client and double checking with everything they said throughout the design phase. This allows me to continue to check myself and make sure that I am keeping to the owner's vision. I also keep standard user practices in mind like intuitive interface and low latency. While not stated int the interview, those need to be considered to create a user-friendly experience. It is important to keep these needs in mind because at the end of the day, the person using your program is the user. If their needs aren't addressed, then you have failed as a developer.

I approach designing software systematically. I go through each phase of the SDLC to create robust systems based on analysis and design. This allows the systems that I make to be cleaner and more maintainable.
